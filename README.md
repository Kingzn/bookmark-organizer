# 浏览器书签自动分类整理应用

一个本地优先的网页版书签整理工具，用于导入浏览器导出的书签 HTML，按链接内容自动分类，检查无效链接，并导出可重新导入浏览器的书签栏文件。

## 功能

- 导入浏览器书签 HTML 文件。
- 先填写行业和职业，用作分类权重。
- 自动分类执行两遍：
  - 第一遍读取标题、域名、URL、网页标题、描述和关键词。
  - 第二遍读取页面正文文案和高频关键词，复核低置信度结果，尽量减少“待整理”。
- 可选使用本地免费大语言模型复核难分类书签。
  - 默认关闭。
  - 需要本机运行 Ollama。
  - 默认模型：`qwen2.5:7b`，可通过 `OLLAMA_MODEL` 修改。
- 检查无效书签，并由用户确认是否删除。
- 导出标准浏览器书签 HTML，顶层为“收藏夹栏”，可导回浏览器作为新的书签栏结构。

## 本地运行

需要 Node.js 18 或更高版本。

```bash
npm start
```

默认访问：

```text
http://localhost:4173
```

如果要指定端口：

```bash
PORT=3000 npm start
```

如果要指定默认书签文件：

```bash
BOOKMARK_FILE=/path/to/bookmarks.html npm start
```

未配置 `BOOKMARK_FILE` 时，页面仍可通过“选择书签 HTML”导入文件；“载入当前书签包”会提示没有默认文件。

## 本地大模型复核

这是可选功能。先安装并运行 Ollama，然后拉取模型：

```bash
ollama pull qwen2.5:7b
```

启动应用后，在页面里勾选“对难分类书签使用本地免费大模型”。

换模型：

```bash
OLLAMA_MODEL=qwen2.5:3b npm start
```

## 导出后导回浏览器

导出的文件名类似：

```text
bookmarks_bar_organized_2026-05-31.html
```

浏览器导入通常只会新增书签，不会自动删除旧书签栏。要替换原有书签栏，建议：

1. 先在浏览器书签管理器中备份现有书签。
2. 清空旧的书签栏内容。
3. 导入本应用导出的 HTML 文件。

## 互联网分享和部署

这个应用包含后端服务，因为需要服务端读取网页、检查链接状态、绕开浏览器跨域限制。所以不建议只用 GitHub Pages，GitHub Pages 只能托管静态页面，链接检测和页面正文分析功能会受限。

推荐部署到支持 Node.js 的平台：

- Render
- Railway
- Fly.io
- Northflank
- 自己的 VPS

通用启动命令：

```bash
npm start
```

云平台环境变量建议：

```text
HOST=0.0.0.0
PORT=平台自动提供或你指定的端口
```

部署后，把平台生成的网址发给朋友即可测试。朋友可以在网页中上传自己的书签 HTML 文件，不需要访问你电脑上的本地书签文件。
